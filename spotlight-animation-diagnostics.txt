/* ──────────────────────────────────────────────────────────────────────
   Slide visibility driven by the box's `data-active-slide` attribute,
   NOT the `.active` class.

   Why: the split-showcase JS sets `data-active-slide` on the box
   synchronously, but the `.active` class doesn't leave the old slide
   until the next requestAnimationFrame (~16ms later).  During that
   window BOTH the old and new slides have `.active` — and the
   split-showcase CSS makes both `position: relative; opacity: 1` →
   they stack in the layout for one frame, visibly showing two slides
   bleeding into each other.

   The data attribute is changed BEFORE the class swap, so binding
   visibility to it kills the overlap instantly.  Outgoing slide
   becomes invisible + position:absolute the moment the JS calls
   `setAttribute('data-active-slide', id)`, no waiting on RAF. */

  opacity: 0 !important;
  visibility: hidden !important;
  position: absolute !important;
  inset: 0 !important;
  pointer-events: none !important;
  transform: none !important;
  transition: none !important;
}
